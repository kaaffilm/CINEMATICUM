# CINEMATICUM — Object Registry

PR8 adds a machine-readable registry for the repository object perimeter.

The registry does not issue a film.

The registry does not make the case release-candidate-ready.

The registry does not admit media.

The registry does not execute replay.

The registry does not produce an admissibility verdict.

## Registry owners

- `CINEMATICUM_OBJECT_REGISTRY.json`
- `CINEMATICUM_SURFACE_CLASS_CATALOG.json`
- `CINEMATICUM_CURRENT_STATE_INDEX.json`

## Purpose

The object registry prevents the public surface from becoming a pile of convincing but ambiguous JSON files.

Every critical object must be readable as one of:

- law object
- schema object
- active current-state object
- layer-status record
- case record
- verification manifest
- progression graph
